# TaskAPI — Agent Configuration

This file is **intentionally empty** as part of the workshop exercise. Your task is to write an `AGENTS.md` for the TaskAPI project.

## Workshop Exercise

Using the TaskAPI project files as reference, create an AGENTS.md that defines:

1. **Project description** — What is TaskAPI and what problem does it solve?
2. **Key roles** — What agent types would be useful for this project? (e.g., code reviewer, test writer, documentation writer)
3. **Decision criteria** — When should different agents be invoked?
4. **Communication patterns** — How should agents coordinate?

## Tips

- Look at other project AGENTS.md files in the shared directory for examples
- Think about the development workflow: a developer might write code, another might review it, another might improve tests
- Consider what makes sense for a small REST API project

## Example Structure

Your AGENTS.md might include:

- **@developer** — Writes feature code
- **@reviewer** — Reviews code quality and consistency
- **@tester** — Expands test coverage
- **@fixer** — Addresses bugs and fixes issues
